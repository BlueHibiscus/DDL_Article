Understanding DDL Commands Of SQL Server With Examples

What is SQL?

Structured Query Language (SQL) is a programming language developed in the 1970s by IBM researchers. It enables creation and manipulation of databases and uses certain commands to carry out the operations designed to facilitate the interaction between databases. SQL statements are classified in various ways which are grouped into sublanguages. The term “Sublanguage” was first brought out by E.F. Codd in 1970 that refers to a computer language used to define or manipulate the structure and contents of a Relational database management system (RDBMS). Sublanguages in SQL is a fascinating technology. Once you get a hang of it, you will no doubt see its value. I, however, in this brief article am specifically picking the DDL aspect of the Sub-language, hope this makes your understanding easier with the examples used.

DDL (DATA DEFINITION LANGUAGE): It consists of SQL commands that are used to define the database and schema. DDL commands are Create, Alter, SP_Rename, Truncate, and Drop.

CREATE:

command creates databases or objects within the database. Objects refer to the tables, indexes, views, synonyms, sequence, stored procedures, functions, and triggers.

Ex: Create a user-defined database named as SQLSTUDIES

Syntax: Create database

Ex: Create our first table EMPLOYEE in SQLSTUDIES database.

Syntax: Create table ([colname_1][datatype][size], [colname_2][datatype[[size], …………….. [colname_N][datatype][size])

CASE1: Create a table “EMPLOYEE” with three columns EMPID, ENAME, and SAL. In the Relational database, the table is called an Entity; columns are the Attributes.

We have created the EMPLOYEE table permanently in the database along with the three columns EID, ENAME, and SAL with their respective data types and data sizes. Let us dig in further and have a look at the table definition of the EMPLOYEE table. In the picture below, we see the usage of the 'Sp_Help' command. CASE2: Create a Table "STUDENTS" with Column Names as SID, SNAME, MATHS, PHYSICS, CHEMISTRY, TOTAL, AVERAGE.

Requirements:

1) Self-generated number incrementation in column SID. A number must begin with 100 and auto-increment by 1.

2) Columns TOTAL and AVERAGE should be auto-calculated by the values in columns Math, Physics, and Chemistry.

The script to write a Create query is a little different from our previous example. Create a table and columns along with data type and data size for a regular column. Sid, Total and Average columns have an additional keyword as shown in the example above.

We can see the number in column SID has begun with 100 and is incrementing by 1. Let us also have a look at different styles of columns supported by SQL Server.

Standard Column: It is a regular column with column name and data type created by the user. For example, in the table 'EMPLOYEE', SNAME. MATH, PHYSICS, and CHEMISTRY are standard columns.

Ex: SNAME Varchar(20)

Computed column: It is a virtual column not physically stored in the table. A Computed column can use data from other columns to calculate a value for the column to which it belongs. For example, in the EMPLOYEE table, the value in each row of the TOTAL and AVERAGE column is dependent on values in MATHS, PHYSICS, and CHEMISTRY columns.

Ex: Total AS (MATH + PHYSICS + CHEMISTRY)

Identity columns: Identity column creates sequentially auto-generated increment numbers. For example, in the STUDENTS table, the number generation for column SID INT IDENTITY (100,1) is automatically generated starting from 100 and incrementing by 1.

Ex : SID int Identity (100,1)

In the example, the first parameter is a seed (100) and the second parameter is increment (1). You must specify both seed and increment or neither. If neither is specified then by default it is (1, 1).

Note:

1) A table can have only one identity column.

2) Identity column only supports number datatypes.

ALTER:

This command is used to modify the structure of the table, column/s or to change the logic of any object.

Alter command is used to -

1). Add a new column(s) in an existing table.

2). Drop a column(s) from an existing table.

3). Change the data type.

4). Change the data size.

To add a new column(s)

Syntax: ALTER TABLE ADD [size]…...

Ex: ( To add a new column JOB in an already existing EMPLOYEE table)

ALTER TABLE EMPLOYEE ADD JOB VARCHAR(30)

Ex: (To add multiple columns EMOBILE, EADDRESS in an already existing EMPLOYEE table)

ALTER TABLE EMPLOYEE ADD EMOBILE INT, EADDRESS VARCHAR (60)

Alter table by dropping a column(s)

Syntax: ALTER TABLE DROP COLUMN

Ex: Drop JOB column from EMPLOYEE table. We see JOB column is removed from EMPLOYEE table.

Alter column from one data type to other

Syntax: ALTER TABLE ALTER COLUMN

Ex: (To change the data type of a SAL column from money to integer data type)

ALTER TABLE EMPLOYEE ALTER COLUMN SAL INT

Before the ALTER command the data type of SAL column was integer, which now changed to money data type.

Alter column data size

Syntax: ALTER TABLE ALTER

Ex: (To change the data size of column Ename Varchar(20) to Ename Varchar(50))

ALTER TABLE EMPLOYEE ALTER COLUMN ENAME VARCHAR(50)

SP_RENAME:

This DDL command is used to rename the column or a table.

Syntax to change the column name:

SP_RENAME [TABLENAME.OLD_COLUMNNAME’],[’NEW_COLUMNNAME’]

Ex: (To change SAL column name to SALARY)

Syntax to change the name of the table:

SP_RENAME [‘OLD_TABLENAME’] ,[ ‘NEW_TABLENAME’]

Ex: To change the name of table from EMPLOYEE to DUPLICATE_EMPLOYEE

In this example, the table name EMPLOYEE has changed to DUPLICATE_EMPLOYEE. The object name EMPLOYEE does not exist in the database anymore. Now, you must address the table name with DUPLICATE_EMPLOYEE, or you will receive an error message.

TRUNCATE:

This command deletes all records from the table in one shot.

Syntax: TRUNCATE TABLE

Ex: TRUNCATE all records from the EMPLOYEE table.

Before executing the TRUNCATE command, the records residing in the EMPLOYEE table are shown.

After executing the TRUNCATE command, all records from EMPLOYEE table are deleted in one go and we are left with empty columns.

DROP:

This command drops/removes the table and all objects associated with it, for example - table structure, records, and dependent objects such as views, constraints, indexes and more in the database.

Syntax: DROP TABLE

Ex: To drop or remove the table EMPLOYEE from the database

Here, we see that the table EMPLOYEE has successfully dropped, which implies that the EMPLOYEE table does not exist in the SQLSTUDIES database anymore. Let us give it a try and execute a SELECT query.

The error message clearly states that the object has ceased to exist and is not valid anymore. The table EMPLOYEE is no more in the database. Drop command is pretty powerful, so use it carefully.
